# AniseKBD
keyboard family to support any split 

---------------

Split keyboards are the real saver for people suffering RSI, but existing split keyboard design are way less than split keyboard design space. 

AniseKBD aims to support any possible split, by design and implement:

0. A general split keyboard description format(DESC for short), and related GUI generation tool
1. keyboard controller circuit board which use BoardToBoard Conn to connect keymatrix circuit board.
2. keyboard case generator, which takes DESC and options(Top/Gasket etc.) as input and generate .stl/.stp file for 3D print or CNC
3. keyboard circuit generator, which takes DESC and options(Hotswap/led postion etc.) as input and generate keymatrix circuit board
4. keyboard firware generator, which takes DESC and options(keymap, led effect etc.) as input and generator firmware files or bin

---------------- 

Keyboards generated by anise flow:
- anise60b: gh60 ANSI layout with 2u space on each side, b stands for basic or boat(bottom row keys are shifted towards center)
- anise60bnav: anise60d witch navigation keys on bottom right.
- (in progress)anise60g: gh60 ANSI layout, witch 2.25u and 2.75u space on left/right side.
- (in progress)anise60eu: gh60 ISO layout to support big RET key and short left shift
- (in progress)anise60hh: happy hacking keyboard like
- (in progress)anise60mi: minila like
- (in progress)anise60mj: minila with large RET and more bottom row keys for kana and etc.
- (drafting) anise75: 6 rows anise60 with an extra column on left right and FN keys
- (drafting) anise-numpad: 21 keys numpad

Keyboards cannot describe but current DESC, need to extend description format to support rotated keys
- (nearfuture) anise-ergodox: ergodox like layout
- (nearfuture) anise-corne: ergodox like layout
